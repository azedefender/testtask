# Проверка баланса игрового рынка (поиск отрицательного цикла)

Студия разработки компьютерных игр создала игру. Геймдизайнер Джон проводит тест баланса для новой игры.
В ней на рынке предметов можно обменять свой уникальный предмет (который у игрока может быть только один)
на другой, причём из-за разницы полезности предметов одна из сторон будет вынуждена доплатить золотом.
После долгого процесса разработки он не уверен, что на рынке предметов нельзя получить золото с
сохранением изначального предмета. Помогите Джону проверить возможность такого случая.

## Условие задачи

Напишите программу на языке PHP, которая определяет, существует ли в системе обменов
«уязвимое место» (дисбаланс) – последовательность обменов, начинающаяся и заканчивающаяся
одним и тем же предметом, после совершения которой у игрока остаётся тот же предмет,
но суммарное количество золота увеличивается (т.е. игрок получает золото).

**Ограничения реализации:**
- Запрещено использовать алгоритм Беллмана–Форда.
- Запрещено использовать ChatGPT (требование к разработчику).

## Формат входных данных

Программа получает данные через стандартный ввод (stdin).

- Первая строка: два целых числа `n` и `m` – количество предметов (вершин) и количество возможных обменов (рёбер).
- Далее `m` строк, каждая содержит три целых числа `u`, `v`, `w` – обмен предмета `u` на предмет `v`, при котором игрок **отдаёт** `w` золота.  
  Если при обмене игрок **получает** золото, `w` отрицательное.

## Формат выходных данных

- Если дисбаланс (отрицательный цикл) обнаружен:
  ```
  YES
  последовательность_обменов
  ```
  Последовательность обменов выводится как номера предметов через пробел, начиная с предмета с минимальным номером в цикле. Последовательность заканчивается тем же предметом, с которого начинается (цикл замкнут).

- Если дисбаланс отсутствует:
  ```
  NO
  ```

### Пример 1 (дисбаланс есть)
**Вход:**

```

```

**Выход:**

```
NO
```

## Используемый алгоритм

Вместо классического для поиска отрицательных циклов алгоритма Беллмана–Форда здесь применён
**алгоритм Флойда–Уоршелла** (Floyd–Warshall). Он строит матрицу кратчайших расстояний между
всеми парами вершин за `O(n³)`. Отрицательный цикл существует тогда и только тогда, когда
для некоторой вершины `i` выполняется `dist[i][i] < 0`.

**Преимущества** данного подхода:
- Полностью соблюдается ограничение задачи (не используется Беллман–Форд).
- Восстановление самого цикла выполняется просто: зная, что из `i` можно дойти до `i` с отрицательным суммарным весом, мы идём по указателям `next` и выделяем замкнутый путь.
- Код легко читается и не требует дополнительного кода для поиска компонент циклов.

**Восстановление цикла:**
1. Находим любую вершину `start`, для которой `dist[start][start] < 0`.
2. Делаем до `n` шагов по матрице `next[текущая][start]`, чтобы гарантированно оказаться внутри цикла.
3. Двигаемся оттуда, запоминая вершины, пока снова не попадём в уже посещённую.
4. Находим минимальную вершину в полученном цикле и выводим цикл, начиная с неё, с повторением первой вершины в конце.

## Как запустить

Склонируйте репозиторий и выполните скрипт в консоли, подав входные данные через stdin:

```bash
php solution.php < input.txt
```

Или с ручным вводом:

```bash
php solution.php
```

## Оценка сложности

- **Время:** `O(n³ + m)`, где `n` – количество предметов, `m` – количество обменов.
- **Память:** `O(n²)`.

При `n` до ~200 программа работает в пределах нескольких сотен миллисекунд, что достаточно для типовых тестовых ограничений.

## Примечания

- При наличии нескольких рёбер между одной и той же парой предметов сохраняется наименьший вес, так как он наиболее выгоден для образования отрицательного цикла.
- Если отрицательных циклов несколько, выводится тот, который был найден первым во время перебора вершин (по порядку номеров), что допустимо.
- Для предотвращения переполнения используется константа `PHP_INT_MAX / 4` в качестве «бесконечности».
